1. Packet Tracer - Navigating The IOS

  1. Part 1 : Establish Basic Connections, Access the CLI, and Explore Help

    1. Langkah 1 : Menghubungkan PC 1 ke S1 (Switch 1) menggunakan kabel konsol

      1. Klik pada icon connections di pojok kiri bawah aplikasi packet tracer, seperti berikut:

      2. Pilih kabel console berwarna biru muda, seperti berikut:

      3. Klik PC1 Maka akan muncul menu dialog, kemudian pilih koneksi RS-232, seperti berikut:

      4. Tarik dari PC1 ke S1 untuk menghubungkan koneksi PC dengan switch:

      5. Muncul dialog menu, lalu pilih port console untuk membuat koneksi baru.

      6. Jika berhasil maka hasilnya akan seperti berikut:

    1. Langkah 2 : Membuat sesi terminal dengan S1

      1. Klik PC1 dan kemudian pilih tab Desktop seperti berikut:

      2. Pilih menu Terminal untuk memeriksa bahwa konfigurasi port sudah sesuai.

      3. Klik tombol OK seperti berikut:

      4. Maka akan tampil jendela terminal, lalu tekan ENTER pada keyboard

    1. Langkah 3 : Telusur Bantuan IOS

      1. IOS dapat memberikan bantuan seperti perintah tergantung pada level yang sedang diakses. Prompt yang sedang tampil saat ini adalah User EXEC, dan sedang menunggu perintah untuk dimasukkan. Pada prompt ketikan symbol tanda tanya (?) untuk menampilkan daftar perintah.

      2. Pada prompt ketik “t” lalu tanda tanya (?)

      3. Pada prompt, ketik “te” dan kemudian tanda tanya (?)

        Jenis bantuan dikenal sebagai Konteks-sensitive

  2. Part 2 : Explore EXEC Modes

    1. Langkah 1 : Masuk ke mode EXEC atau mode Khusus

      1. Pada prompt, ketik tanda Tanya (?) seperti berikut:

      2. Ketik “en” dan tekan tombol Tab

        Ini disebut juga dengan penyempurnaan perintah (penyempurnaan tab). Ketika bagian perintah diketik, Tabdapat digunakan untuk melengkapi Sebagian perintah.

      3. Ketik perintah “enable” dan tekan enter

      4. Pada Prompt, ketik tanda tany kembali (?)

    2. Langkah 2 : Masuk ke mode konfigurasi global

      1. Saat berada dalam mode EXEC yang khusus, perintah yang dimulai dengan huruf ‘C’ adalah configure. Ketikan perintah lengkap atau perintah Sebagian untuk membuatnya unik. Tekan tombol Tab untuk mengeluarkan perintah dan tekan ENTER.

      2. Tekan Enter untuk menerima parameter default diantara tanda kurung [terminal]

      3. Mode Konfigurasi global. Mode ini akan ditelusuri lebih lanjut dalam aktivitas kedepanya, untuk saat ini Kembali ke mode EXEC dengan mengetikan end, exit, atau CTRL+Z.

  3. Part 3 : Set the Clock

    1. Langkah 1 : Menggunakan perintah “clock”

      1. Gunakan perintah clock untuk melihat lebih lanjut bantuan dan sintaks dari perintah. Ketikan show clock pada prompt EXEC khusus.

      2. Gunakan perintah sensitive context dan perintah clock untuk mengatur waktu pada switch ke waktu saat ini. Masukkan perintah clock dan tekan ENTER.

      3. Muncul “%inclompete command”. Artinya kita perlu perintah parameter untuk perintah jam tersebut. Ketikan perintah clock dan tambahan tanda tanya (?)

      4. Atur jam menggunakan perintah clock set. Lanjutkan melalui perintah tersebut pada perintah waktu.

      5. Berdasarkan informasi yang diminta pada perintah clock set? Masukkan perintah waktu 19:00 P.M dengan menggunakan satuan format 24 Jam.

      6. Ketikan clockset 19:00:00 31 Jan 2035. Lalu lihat jam yang sudah disetting dengan show clock.

    2. Langkah 2 : Telusuri pesan perintah tambahan

      1. Tetap gunakan perintah clock untuk menelusuri perintah tambahanya.

      2. Ketikkan perintah berikut:




2. Packet Tracer - Configuring Initial Switch Settings

  1. Part 1 : Verify the Default Switch Configuration

    1. Langkah 1 : Masuk ke mode khusus Exec.

      1. Klik S1 kemudian tab CLI. Tekan enter.

      2. Masuk ke mode khusus EXEC istimewa dengan menginputkan perintah enable.

    2. Langkah 2 : Memeriksa konfigurasi switch.

      1. Masukkan perintah Show Running-Config

      2. Jawaban:

        1. Antarmuka FastEthernet yang dimiliki switch tersebut ada 24.
        2. Antarmuka Gigabit Ethernet yang dimiliki switch ada 2
        3. Value pada line vty 0-4 dan vty 5-15
        4. Perintah #show startup-config
        5. Karena konfigurasi startup belum disimpan

  2. Part 2 : Configure a Basic Switch Configuration

    1. Langkah 1 : Menetapkan nama switch.
    2. Langkah 2 : Mengamankan akses ke jalur console.
    3. Langkah 3 : Memverifikasi akses console aman.

      Keluar dari mode khusus untuk memverifikasi bahwa kata sandi port console berlaku.

    4. Langkah 4 : Mengamankan hak khusus pada mode akses.
    5. Langkah 5 : Memverifikasi hak mode akses khusus aman.

      1. Masukkan perintah exit lagi untuk keluar dari switch

      2. Tekan Enter dan sekarang akan memasukkan kata sandi:

      3. Masukkan perintah berikut untuk mengakses mode khusus:

      4. Masukkan kata sandi kedua yang sudah dikonfigurasi untuk melindungi hak khusus di mode EXEC:

      5. Verifikasi konfigurasi dengan memeriksa isi file running-configuration

    6. Langkah 6 : Konfigurasi kata sandi terenkripsi untuk mengamankan akses ke mode khusus.

      Mengkonfigurasi kata sandi rahasia untuk masuk ke mode EXEC.

    7. Langkah 7 : Verifikasi aktifkan kata sandi rahasia pada file konfigurasi.

      Masukkan kembali perintah #show run untuk memverifikasi. Karena ini merupakan kata sandi yang dienkripsi secara otomatis.

    8. Langkah 8 : Enkripsi kata sandi dan kata sandi console.

  3. Part 3 : Configure a MOTD Banner

    1. Langkah 1 : Konfigurasi pesan hari ini (MOTD)
      1. Banner MOTD ini akan ditampilkan saat keluar dari mode khusus.
      2. Agar mengetahui setiap switch itu memiliki peran yang berbeda tergantung dari pemakaiannya.

  4. Part 4 : Save Configuration Files to NVRAM

    1. Langkah 1 : Pastikan bahwa konfigurasi akurat menggunakan perintah show.
    2. Langkah 2 : Simpan File konfigurasi

      Versi terpendek dari copy running-config startup-config adalah copy run start.

    3. Langkah 3 : Periksa file konfigurasi startup.

      - Perintah #show startup-configurasi

  5. Part 5 : Configure S2

    1. Konfigurasi S2 menggunakan konfigurasi dengan parameter sebagai berikut:

      1. Nama perangkat S2.

      2. Lindungi akses console menggunakan kata sandi letmein.

      3. Konfigurasikan kata sandi dengan c1$c0 dan kata sandi rahasia itssecret.

        Konfigurasikan pesan kepada user yang masuk ke switch dengan pesan berikut : “Authorized access only, Unauthorized access is prohibited and violators will be prosecuted to the full extent of the law”.

      4. Enkripsi semua kata sandi teks biasa.

      5. Simpan dalam file konfigurasi menghindari kehilangan jika switch dimatikan.

3. Packet Tracer - Configure Initial Router Settings

  1. Part 1 : Verify the Default Router Configuration

    1. Langkah 1 : Buat koneksi console ke R1.

      1. Pilih kabel console dari koneksi

      2. Klik PCA dan pilih RS232

      3. Klik R1 lalu pilih console

      4. Klik PCA, kekmudian pilih pada tab Desktop dan pilih terminal.

      5. Klik OK, lalu tekan enter. Selanjutnya melakukan konfigurasi R1.

    2. Langkah 2 : Masuk ke mode khusus dan periksa konfigurasi saat ini.

      1. Masuk ke mode khusus EXEC dengan masukkan perintah enable.

      2. Ketikan perintah show running-config.

      3. Jawaban:

        1. Router
        2. Ada 4 antarmuka FastEthernet
        3. Ada 2 antarmuka Gigabit
        4. Ada 2 Antarmuka serial
        5. Line vty 0-4

      4. Tampilkan NVRAM saat ini:

        Pesan ini karena file konfigurasi tidak disimpan ke NVRAM

  2. Part 2 : Configure and Verify the Initial Router Configuration

    1. Langkah 1 : Konfigurasikan pengaturan awal pada R1.

      1. R1 sebagai nama host

      2. Menggunakan kata sandi sebagai berikut:

        1. Console: letmein

        2. Hak khusus di EXEC, tidak terenkripsi: cisco

        3. EXEC yang khusus, dienkripsi: itsasecret

      3. Enkripsi semua kata sandi teks biasa

      4. Banner MOTD: “unauthorized access is strictly prohibited”


    2. Langkah 2 : Verifikasi pengaturan awal R1.

      1. Verifikasi pengaturan awal dengan melihat konfigurasi f atau R1

      2. Keluar dari sesi console saat ini.

      3. Tekan enter dan akan melihat pesan berikut.

        Karena setiap router harus mempunyai pesan yang menginisialkan dari setiap router tersebut sesuai dengan kebutuhan

      4. Masukkan kata sandi yang diperlukan untuk kembali ke mode EXEC khusus.


  3. Part 3 : Save the Running Configuration File

    1. Langkah 1 : Simpan file konfigurasi ke NVRAM

      1. Sekarang membuat cadangan file konfigurasi yang sedang berjalan ke NVRAM dengan menyimpannya.

        - Versi terpendek dan tidak ambigu dari perintah ini adalah copy r s.

        - Perintah yang menampilkan isi NVRAM show start.

      2. Verifikasi bahwa semua parameter yang dikonfigurasikan telah disimpan.

    2. Langkah 2 : Simpan konfigurasi startup ke flash

      1. Memeriksa isi flash menggunakan show flash

      2. Simpan file konfigurasi startup ke flash

      3. Gunakan perintah show flash untuk memverifikasi file konfigurasi startup sekarang disimpan dalam flash

  1. Bagian 1 : Menampilkan Informasi Router

    1. Langkah 1 : Tampilkan informasi antarmuka R1

      1. Perintah yang menampilkan statistic untuk semua antarmuka konfigurasi pada router

      2. Perintah yang menampilkan informasi tentang serial antarmuka 0/0/0

      3. Masukkan perintah untuk menampilkan statistic antarmuka serial 0/0/0 pada R1

        1. Alamat IP yang dikonfigurasi pada R1 > 209.165.200.225/30
        2. Bandwidth pada antarmuka serial 0/0/0 > 1544 Kbits

      4. Masukkan perintah untuk menampilkan statistic antarmuka GigabitEthernet 0/0

        1. IP pada R1 untuk GigabitEthernet tidak ada konfigurasinya
        2. MAC address dari antarmuka GigabitEthernet 0/0 > 1000000 kbit
    2. Langkah 2 : Tampilkan daftar ringkasan antarmuka pada R1

      1. Perintah yang menampilkan ringkasan singkat dari interfaces saat ini, status, dan alamat IP

      2. Masukkan perintah pada setiap router:

        1. Berapa banyak antarmuka serial yang ada di R1 dan R2 -> Setiap router memiliki 2 antarmuka serial
        2. Berapa banyak antarmuka Ethernet yang ada di R1 dan R2 -> R1 memiliki 6 antarmuka Ethernet sedangkan R2 hanya 2 antarmuka Ethernet
        3. Apakah semua antarmuka Ethernet pada R1 sama? -> Tidak. Ada 2 antarmuka Gigabit Ethernet dan 4 antarmuka Fast Ethernet. Antarmuka Gigabit Ethernet mendukung kecepatan hingga 1.000.000.000 bit dan antarmuka Fast Ethernet mendukung kecepatan hingga 1.000.000 bit
    3. Langkah 3 : Tampilkan tabel routing R1

      1. Perintah apa yang menampilkan isi tabel routing

      2. Masukkan perintah pada R1 :

        1. Berapa banyak rute yang terhubung (menggunakan kode C) > 1
        2. Rute yang terlist > 209.165.200.224/30 dan 209.165.200.225/30
        3. Bagaimana cara router menangani paket yang ditujukan untuk jaringan yang tidak tercantum dalam tabel rute -> Router hanya akan mengirim paket ke jaringan yang terdaftar di tabel routing. Jika jaringan tidak terdaftar, paket akan keluar.

  2. Bagian 2 : Konfigurasi Antarmuka Router

    1. Langkah 1 : Konfigurasi antarmuka GigabitEthernet 0/0 pada R1

      1. Masukkan perintah berikut ke alamat dan aktifkan antarmuka GigabitEthernet 0/0 pada R1

      2. Konfigurasi deskripsi antarmuka yang menunjukan kemana perangkat terhubung

      3. Sekarang R1 dapat melakukan ping ke PC1

    2. Langkah 2 : Konfigurasi antarmuka GigabitEthernet yang tersisa di R1 dan R2

      1. Gunakan informasi di tabel pengalamatan untuk konfigurasi antarmuka untuk R1 dan R2

        1. Masukkan alamat IP dan aktifkan antarmuka
        2. Konfigurasi deskripsi yang sesuai

      2. Verifikasi Konfigurasi antarmuka


    3. Langkah 3 : Cadangkan Konfigurasi ke NVRAM

      Menyimpan file konfigurasi di kedua router ke NVRAM > copy run start

  3. Bagian 3 : Verifikasi Konfigurasi

    1. Langkah 1 : Gunakan perintah verifikasi untuk memeriksa konfigurasi antarmuka

      1. Gunakan perintah show ip interface brief pada R1 dan R2

        1. Berapa banyak antarmuka pada R1 dan R2? > 3 pada setiap router
        2. Bagian mana dari konfigurasi antarmuka yang TIDAK ditampilkan dalam output? > Subnetmask
        3. Perintah apa yang digunakan untuk memverifikasi bagian konfigurasi > show interfaces, show ip protocols, show run

      2. Gunakan perintah show ip route pada R1 dan R2 untuk melihat tabel routing

        1. Berapa banyak rute yang terhubung (menggunakan C kode) yang terlihat disetiap router? 3
        2. Berapa banyak rule EIGRP (menggunakan kode D) yang terlihat pada setiap router? 2
        3. Jika router mengetahui semua rute dalam jaringan, maka jumlah rute yang terhubung dan rute yang dipelajari secara dinamis (EIGRP) harus sama dengan jumlah total LAN dan WAN. Berapa banyak LAN dan WAN dalam topologi? 5
        4. Apakah nomor ini tersedia dengan jumlah rute C dan D yang ditunjukkan pada tabel routing? Ya
    2. Langkah 2 : Uji Konektivitas ujung ke ujung pada seluruh jaringan

      1. Dari bari perintah di PC1, ping PC4

      2. Dari baris perintah di R2, ping ke PC2

Demikian pembuatan jaringan sederhana 2 menggunakan Cisco Packet Tracer, terimakasih.
